# MVP 웹사이트 명세서 (유형 진단 전용)

## 문서 목적

이 문서는 바이브 코딩 AI가 **가장 빠르게 만들 수 있는 첫 번째 검증용 제품**을 만드는 데 필요한 명세서다.
범위는 넓지 않다. 이 버전은 **웹 기반의 단일 경험**에 집중한다.

즉, 이 MVP는 아래 한 가지를 검증하기 위한 것이다.

> 사용자가 짧은 진단을 통해 “이 서비스가 나를 꽤 잘 읽는다”는 감각을 느끼는가?

이 문서는 “실제 앱” 전체가 아니라,

* 브랜드 첫인상
* 짧은 진단 경험
* 결과 해석
* 자신의 유형 확인
  이 네 가지를 검증하는 **진단 중심 웹사이트**에 대한 명세서다.

---

# 1. 이 MVP의 목표

## 핵심 목표

이 MVP 웹사이트의 목표는 다음 세 가지다.

1. 사용자가 서비스의 방향을 **즉시 이해**하게 만든다.
2. 사용자가 짧은 진단을 통해 **자기 유형/자기관리 성향**을 받아보게 한다.
3. 사용자가 결과를 보고 **“이거 내 얘기네”**라고 느끼게 만든다.

## 이 MVP가 아직 하지 않는 것

이 버전에서는 다음은 하지 않는다.

* 정식 계정 가입
* 결제
* 복잡한 루틴 관리
* 장기 추적
* 알림 시스템
* 고급 AI 코칭
* 커뮤니티

즉, 첫 MVP는 **진단형 웹 경험**에만 집중한다.

---

# 2. 제품 한 줄 정의

이 웹사이트는 **당신에게 맞는 자기관리 방식과 유형을 진단해주는 웹 경험**이다.

더 정확히 말하면,

* 단순 성격 테스트가 아니다.
* 멘탈 회복 도구도 아니다.
* 자기계발 자극 콘텐츠도 아니다.

이 경험은 사용자가

* 어떤 식으로 자기관리를 해야 잘 맞는지
* 어떤 방식일 때 오래 가는지
* 어떤 유형의 운영 방식이 본인에게 맞는지
  를 빠르게 받아보게 하는 것이 목적이다.

---

# 3. 브랜드/디자인 방향 (매우 중요)

이 MVP는 “기능은 단순하지만 디자인은 매우 좋아 보여야 한다.”

## 디자인 목표

사용자가 첫 화면을 보자마자 느껴야 하는 인상:

* 이건 흔한 MBTI 테스트 사이트가 아니다
* 이건 싸구려 자기계발 계정도 아니다
* 이건 훨씬 더 성인 취향이고, 미감이 있고, 정리된 서비스 같다
* 단순하지만 수준이 있다

## 참고 무드

이전 레퍼런스 이미지에서 가져와야 할 결은 아래와 같다.

* 블랙/화이트/그레이 중심의 모노톤
* 강한 타이포그래피
* 넓은 여백
* 잡지 화보/에디토리얼 같은 레이아웃 감각
* “정돈된 남성성”이 느껴지는 분위기
* 과하지 않은 긴장감
* 무겁지만 촌스럽지 않은 화면

## 절대 피할 것

* 빨간색 강조, 불꽃, 과한 동기부여 시각 언어
* 근육, 명품 과시, 공격적인 성공학 톤
* MBTI 놀이형 밝고 가벼운 UI
* 생산성 툴처럼 지나치게 밋밋한 SaaS UI
* 상담앱처럼 지나치게 부드럽고 포근한 파스텔 톤

## 시각 시스템 권장

* 배경: 오프화이트 또는 딥 블랙
* 텍스트: 검정 / 흰색 / 짙은 회색
* 포인트 컬러: 아주 절제된 메탈릭 실버 느낌 또는 아주 미세한 차콜 계열
* 버튼: 크고 무겁지 않게, 단정하고 명확하게
* 카드: 모서리는 지나치게 둥글지 않게, 너무 테크 회사 같지도 않게
* 그림자: 과하지 않게
* 애니메이션: 느리고 고급스럽게, 튀지 않게

## 타이포그래피 방향

* 헤드라인은 강하게
* 본문은 절제되게
* 숫자/결과/키워드는 구조감 있게 강조
* 문장 길이는 짧게

---

# 4. 이 MVP에서 사용자가 경험하는 전체 흐름

사용자는 아래 흐름을 따라야 한다.

1. 랜딩 페이지 도착
2. 서비스가 무엇인지 직관적으로 이해
3. “진단 시작하기” 클릭
4. 10~12문항 진단 진행
5. 결과 로딩
6. 자신의 유형/해석/간단한 원칙 확인
7. 결과 저장 또는 베타 대기 등록 유도

이 흐름은 반드시 **3분 안팎**으로 끝나야 한다.

# 5. 정보 구조 / 페이지 구조

## 페이지 1. Landing

### 목적

* 서비스 정체성을 바로 전달
* 사용자를 진단 경험으로 보내기

### 반드시 있어야 하는 요소

* 강한 헤드라인 1개
* 서비스 설명 1~2문장
* 진단 시작 CTA 버튼
* 아주 짧은 보조 설명
* 시각적으로 고급스러운 히어로 섹션

### 헤드라인 방향 예시

* 잘 사는 사람은 자기관리 방식부터 다릅니다
* 꾸준함은 의지가 아니라 방식에서 나옵니다
* 당신에게 맞는 자기관리 방식은 따로 있습니다

### 서브카피 예시

* 짧은 진단으로 당신의 자기관리 유형을 분석합니다
* 루틴, 집중, 생활 습관을 바탕으로 당신에게 맞는 방식을 보여줍니다

### CTA 예시

* 내 유형 진단하기
* 나에게 맞는 방식 찾기
* 3분 진단 시작

## 페이지 2. Diagnosis Intro

### 목적

* 진단 전 기대감을 만든다
* 너무 길지 않게 진입 장벽을 낮춘다

### 포함 요소

* “총 12문항 / 3분 내 완료” 안내
* 진단으로 알 수 있는 것 3가지
* 시작 버튼

예:

* 어떤 방식으로 자기관리를 해야 잘 맞는지
* 어떤 성향일 때 꾸준함이 생기는지
* 어떤 유형의 시스템이 나에게 맞는지

## 페이지 3. Question Flow

### 목적

* 빠르고 매끄럽게 답하게 한다
* 지루하지 않고 고급스럽게 느껴지게 한다

### 요구사항

* 한 화면에 한 문항
* 상단 진행률 표시
* 일부 문항은 복수 선택 가능(최대 2개)
* 이전/다음 가능
* 모바일에서 입력이 편해야 함

### UI 톤

* 큰 질문 문장
* 단정한 선택지 버튼
* 선택 시 반응이 명확해야 함
* 과한 일러스트/아이콘 불필요

## 페이지 4. Loading / Result Transition

### 목적

* 사용자가 “내 결과를 진짜 분석 중”이라는 느낌을 받게 한다

### 화면 문구 예시

* 당신의 답변을 바탕으로 유형을 분석하고 있습니다
* 어떤 방식이 당신에게 잘 맞는지 정리하고 있습니다

### 주의

* 너무 오래 걸리면 안 됨
* 1~2초 정도의 고급스러운 전환이면 충분

## 페이지 5. Result

### 목적

* 핵심 가치 제공
* “이거 내 얘기다” 만들기
* 더 깊은 앱/베타로 이어지는 연결점 만들기

### 결과 페이지 구성

1. 결과명
2. 한 줄 요약
3. 해석 문단
4. 당신에게 맞는 방식 3개
5. 피해야 할 방식 2~3개
6. CTA (결과 저장 / 베타 신청 / 앱 대기 등록)

# 6. 질문 설계 원칙

## 목표

이 진단은 단순한 재미 테스트가 아니라,
사용자의 자기관리 성향과 선호 시스템을 빠르게 읽어야 한다.

## 질문 수

* 12문항 권장

## 형식

* 단일 선택 8문항
* 복수 선택 4문항 (최대 2개)

## 언어 원칙

* 쉬운 생활 언어 사용
* 너무 브랜드 단어를 앞세우지 말 것
* 너무 상담 문항처럼 보이지 말 것
* 읽는 순간 답이 떠오르게 만들 것

## 질문 영역

1. 어떤 삶을 선호하는가
2. 어떤 방식일 때 만족도가 높은가
3. 어떤 자기관리 방식이 잘 맞는가
4. 무엇을 가장 견디기 싫어하는가
5. 꾸준함이 생기는 조건은 무엇인가
6. 자기 통제 / 루틴 / 집중 / 실행 중 어디를 더 원하는가

---

# 7. 진단 질문 예시

## Q1

당신이 가장 원하는 삶의 모습에 가까운 것은 무엇입니까?

* 흔들리지 않는 삶
* 해야 할 일을 정확히 처리하는 삶
* 정리되고 단정한 삶
* 내가 원하는 방향으로 움직이는 삶
  (복수 선택 가능)

## Q2

당신이 가장 높이고 싶은 것은 무엇입니까?

* 꾸준함
* 집중력
* 자기관리
* 자기 통제력
  (복수 선택 가능)

## Q3

당신은 어떤 방식으로 살 때 가장 만족스럽습니까?

* 계획한 걸 지켰을 때
* 하루가 정리되어 있을 때
* 내 삶이 내 통제 안에 있을 때
* 남들보다 더 나은 방식으로 살고 있을 때

## Q4

새로운 변화를 만들 때 당신은 어디서부터 손대는 편입니까?

* 생활 리듬
* 할 일과 계획
* 주변 환경
* 생각과 태도

## Q5

당신이 가장 견디기 싫은 상태는 무엇입니까?

* 느슨해진 상태
* 해야 할 걸 놓치는 상태
* 평범하게 흘러가는 상태
* 내 수준보다 낮게 사는 상태
  (복수 선택 가능)

## Q6

당신이 더 잘 맞는 방식은 어떤 쪽입니까?

* 일정이 정리된 방식
* 바로 실행하는 방식
* 반복 가능한 루틴 중심 방식
* 상황에 맞게 유연하게 움직이는 방식

## Q7

하루가 잘 풀리는 날은 보통 어떤 날입니까?

* 시작이 깔끔한 날
* 해야 할 일이 분명한 날
* 집중이 잘 되는 날
* 통제감이 살아 있는 날

## Q8

꾸준함을 만들 때 더 중요한 것은 무엇입니까?

* 강한 의지
* 분명한 기준
* 나에게 맞는 방식
* 반복 가능한 시스템

## Q9

자기관리를 오래 유지하는 데 더 필요한 것은 무엇이라고 느끼십니까?

* 생활 안정
* 집중 관리
* 실행 구조
* 나에게 맞는 규칙

## Q10

당신은 어떤 말에 더 반응합니까?

* 열심히 하자
* 더 잘 살자
* 제대로 해보자
* 나에게 맞게 해보자

## Q11

지금 당신에게 가장 필요한 것은 무엇입니까?

* 생활을 정리하는 것
* 루틴을 만드는 것
* 더 잘 맞는 자기관리 방식
* 오래 가는 시스템
  (복수 선택 가능)

## Q12

당신에게 더 가까운 표현은 무엇입니까?

* 나는 구조가 있어야 잘한다
* 나는 시작이 쉬워야 잘한다
* 나는 반복 가능해야 오래 간다
* 나는 내 방식이어야 유지된다

# 8. 결과 설계

## 결과 유형 수

* 4개 또는 5개 권장

## 권장 결과 유형 예시

1. 루틴 중심형
2. 실행 구조형
3. 자기 통제형
4. 집중 최적화형
5. 시스템 설계형

## 결과 화면 구조 상세

### A. 결과명

사용자가 이해 가능한 수준의 단어 사용

### B. 핵심 문장

예:

* 당신은 의지보다 구조가 있을 때 더 잘 작동하는 사람입니다.
* 당신은 자극보다 시스템이 있을 때 꾸준함이 만들어지는 사람입니다.

### C. 해석 문단

* 당신은 단순히 더 열심히 살고 싶은 것이 아니라, 더 제대로 살고 싶은 사람입니다.
* 남들이 좋다는 방식보다, 당신에게 맞는 방식이 있어야 오래 갑니다.
* 당신은 반복 가능한 구조가 생길 때 자기관리가 살아나는 편입니다.

### D. 당신에게 맞는 방식

예:

* 시작 마찰이 낮은 루틴
* 짧고 명확한 집중 블록
* 무리 없는 기준선 설정

### E. 잘 맞지 않는 방식

예:

* 과하게 빡센 계획
* 감정에 기대는 동기부여형 접근
* 남들 기준 그대로 따라가는 방식

### F. CTA

* 결과 저장하기
* 앱 오픈 알림 받기
* 베타 대기 등록

# 9. 디자인 상세 요구사항

## 핵심 원칙

이 MVP는 기능보다 **미감과 집중감**이 중요하다.
진단 사이트 같아야 하지만, 흔한 테스트 사이트처럼 보이면 안 된다.

## 홈/랜딩 디자인

* 큰 헤드라인
* 절제된 서브카피
* 큰 CTA
* 매우 여백감 있는 레이아웃
* 텍스트 중심 히어로
* 배경은 단순하지만 깊이감 있게

## 질문 화면 디자인

* 질문 하나에 집중
* 선택지 버튼은 큰 면적
* 버튼 간격 넉넉하게
* 진행률은 선형 또는 숫자형으로 미니멀하게

## 결과 화면 디자인

* 결과명은 크게
* 핵심 문장은 짧게 강하게
* 해석은 읽기 좋게 분절
* 원칙/방식 카드는 구조감 있게 배치
* 공유 가능한 시각 구성 고려

## 이미지 사용 여부

초기 MVP에서는 굳이 사진이 많을 필요는 없다.
하지만 배경 또는 일부 섹션에서 아주 절제된 에디토리얼 감성의 이미지 사용은 가능하다.

권장 방향:

* 흑백 톤
* 정적인 공간
* 남성 라이프스타일을 직접적으로 과시하지 않는 이미지
* 물건/공간/질감 중심도 가능

# 10. 기술 구현 제안

## 권장 형태

* 모바일 우선 웹
* React / Next.js 또는 유사 프론트엔드 프레임워크
* 정적 사이트 + 간단한 상태 관리
* 결과 로직은 규칙 기반

## 필요한 최소 기능

* 질문 응답 저장
* 결과 계산
* 결과 렌더링
* 이메일/대기 등록 폼(optional)

## 로그인

초기에는 불필요

## 결과 로직

각 답변을 4~5개 유형에 가중치로 매핑
가장 높은 스코어의 결과 유형을 반환

필요시 혼합형 결과도 가능하지만, 첫 버전은 단일 대표 유형으로 시작 가능

---

# 11. 성공 기준

이 MVP는 아래를 만족하면 성공이다.

1. 사용자가 첫 화면에서 서비스를 이해한다.
2. 진단을 끝까지 완료한다.
3. 결과를 보고 “내 얘기다”라고 느낀다.
4. 앱 오픈/베타에 관심을 가진다.
5. 디자인이 싸구려 테스트 사이트처럼 보이지 않는다.

---

# 12. 바이브 코딩 AI에게 요청할 산출물

이 문서를 바탕으로 아래를 만들어 달라.

1. 모바일 우선 랜딩 페이지
2. 12문항 질문 플로우
3. 결과 계산 로직(mock 포함 가능)
4. 결과 페이지 4~5종 템플릿
5. 대기 등록 CTA
6. 고급스럽고 에디토리얼한 UI 스타일 적용

# 마지막 요약

이 MVP 웹사이트는 **당신에게 맞는 자기관리 유형을 진단해주는 고급스러운 웹 경험**이어야 한다.
핵심은 단순하다.

* 빠르게 이해되고
* 짧게 진단하고
* 정확히 읽히고
* 브랜드가 좋아 보여야 한다.

이것은 첫 번째 검증용 제품이며, 실제 앱 전체가 아니다.
